Terminator

Terminator is focused on Bug Bounty, AI Security, and Client Pitch workflows.

Supported Modes

./terminator.sh bounty https://target.com "*.target.com"
./terminator.sh ai-security https://app.example.com "LLM agent workflow"
./terminator.sh client-pitch https://company.com
./terminator.sh status
./terminator.sh logs

Default Runtime

The default runtime is scope-first-hybrid. Codex/OMX handles bulk recon, source sweeps, analysis, PoC drafting, critic, and triager simulation. Claude handles scope, safety, reporting, and submission governance.

BackendPrimary roles
Codex/OMXtarget-discovery, scout, recon-scanner, source-auditor, analyst, exploiter, critic, triager-sim
Claudescope-auditor, reporter, submission-review, governance/safety decisions

Pipeline

Recon/Input
  -> Raw Endpoint Inventory
  -> Surface Normalizer
  -> Risk Classifier
  -> Vulnerability Hint Engine
  -> Business Risk Mapper
  -> Risk Score + Confidence Score
  -> Raw Endpoint Review Queue
  -> Safe Test Planner
  -> Output Router

Removed Modes

ctf, firmware, robotics, supplychain, and bounty-explore are archive-only on archive/reference-legacy-modes-pre-bounty-ai.
